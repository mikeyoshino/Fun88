Custom Game Import (Own Games)
Admin form fields:
  Title (EN)             — required
  Description (EN)       — required
  Controls (EN)          — optional
  Game URL / Embed code  — required (iframe src or full embed HTML)
  Thumbnail              — file upload → Supabase Storage
  Categories             — multi-select from existing categories
  Auto-translate to Thai — checkbox (default: checked)

provider_id   → NULL  (signals "own game" throughout the system)
provider_game_id → NULL

After save: same pipeline as scraper
  → thumbnail upload → persist → enqueue Thai translation (if checked)
Quartz.NET Job Architecture
Jobs (all implement IJob, registered via DI):

  ScraperJob          — fetches from GameDistribution, runs pipeline for each game
                        Trigger: cron from scraper_schedules table OR manual admin call
                        Concurrency: DisallowConcurrentExecution attribute
                        State: persisted in quartz DB tables (not memory)

  TranslationJob      — calls OpenAI, writes game_translations row for "th"
                        Trigger: one-shot, fired by pipeline after each game import
                        Input: game_id passed via JobDataMap (no ambient state)
                        Retry: 3 attempts with exponential back-off

Admin on-demand trigger:
  POST /admin/scraper/run  →  IScheduler.TriggerJob(JobKey.ScraperJob)
  Returns job ID → admin polls /admin/scraper/jobs/{id} for status

Duplicate Prevention Strategy

GameDistribution games

Check provider_game_id + provider_id before insert. Upsert strategy: update metadata if game already exists (title, thumbnail may change upstream).

Custom games

Slug uniqueness enforced at DB level. Admin sees a clear error if slug already exists — no silent overwrites.
