Key Architectural Decisions

Data Access

EF Core 8 + PostgreSQL (Supabase). Repository pattern per module. No raw SQL except for performance-critical reporting queries.

Auth

Supabase Auth for public users. ASP.NET Identity-backed cookie auth for admin. Supabase JWT validated in middleware.

Scheduling

Quartz.NET with database job store (persists to PostgreSQL). Admin triggers on-demand via controller → IScheduler API.

Localisation

ASP.NET Core IStringLocalizer for UI strings. Game/blog content stored in *_translations tables, fetched by ISO language code ("en" / "th").

No Magic Strings

All route names, policy names, language codes, cache keys, and config keys are static readonly constants or strongly-typed enums.

File Storage

Supabase Storage for game thumbnails and blog featured images. Typed StorageClient wrapper — no direct SDK calls in controllers.
